## Architecture Overview

**Nuxt 3 fullstack app** â€” Vue 3 frontend + Nitro server-side API, deployed as a single Node process. No separate backend.

### Multi-tenancy

Every data model has an `orgId: String` field referencing `Organization._id`. The super-admin sees all orgs; regular admins/users are scoped to their `orgId`. The `rrhh` Pinia store's `currentOrgId` drives all data fetches â€” it's set from `authStore.user.orgId` (or manually switched by super-admin).

### Authentication flow

1. `POST /api/auth/login` â€” hashes password with SHA-256 (Web Crypto API), generates a 64-char hex token, stores it on the `User` document (`user.token`, `user.tokenExpires`).
2. Client stores `{ token, expires, currentOrgId }` in `localStorage` under key `rrhh_session`.
3. Subsequent API calls send `Authorization: Bearer <token>`.
4. `server/utils/requireAuth.js` looks up the User by token + checks expiry on every protected request.
5. `middleware/auth.js` (client-only) checks localStorage session before any `/rrhh/*` route; redirects to `/login` on miss/expiry.
6. On app mount, `authStore.init()` calls `GET /api/auth/me` to rehydrate the user object from the server.

**Roles:** `admin` (full), `manager` (ver/editar/crear), `viewer` (solo ver). Super-admin = `esSuperAdmin: true` OR `rol: admin` with no `orgId`.

**Seed admin:** On first boot (no users in DB), `server/plugins/mongoose.js` creates `admin@rrhh.cl` / `Admin1234!` with `esSuperAdmin: true`.

### Server API conventions

All routes under `server/api/` use file-based Nitro routing:
- `[resource]/index.get.js` â†’ `GET /api/resource`
- `[resource]/index.post.js` â†’ `POST /api/resource`
- `[resource]/[id].put.js` â†’ `PUT /api/resource/:id`
- `[resource]/[id].delete.js` â†’ `DELETE /api/resource/:id`

Every handler starts with `requireDb(event)` (throws 503 if MongoDB not connected). Protected routes also call `await requireAuth(event)`.

Custom `_id` format: `prefix_timestamp_random` (e.g. `w_1716000000000_abc12`), generated by `newId(prefix)` in `server/utils/db.js`. Never use MongoDB's default ObjectId.

### Billing / Subscriptions (`pages/rrhh/admin/billing.vue`)

Super-admin page at `/rrhh/admin/billing`. Plans: `free` (default) or `paid` ($40 USD/worker/month). Three payment gateway scaffolds are ready â€” activate by setting env vars:

| Gateway | Env vars | Coverage |
|---|---|---|
| Stripe | `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET` | Global USD |
| MercadoPago | `MP_ACCESS_TOKEN` | Chile/LatAm CLP |
| Transbank WebpayPlus | `TB_COMMERCE_CODE`, `TB_API_KEY_SECRET`, `TB_ENV` | Chile CLP only |

CLP conversion uses a static rate of 950 CLP/USD (update in billing page as needed).
